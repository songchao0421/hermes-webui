## 二、新增功能详情

### 2.1 引导页 (Onboarding)

**新增路由：** `GET /api/onboarding/status` — 检测是否为首次启动
**新增路由：** `POST /api/onboarding/complete` — 保存配置，标记引导完成

**检测逻辑：**
- `~/.hermes/hermes-webui/.onboarding_done` 文件是否存在
- 如不存在 → 前端跳转引导页
- 引导完成后写入该文件

**引导步骤：**
1. 自动检测 Ollama（`/api/onboarding/detect-ollama` → 扫描 localhost:11434）
2. 可选远程 API 配置（DeepSeek/OpenAI/OpenRouter + 验证余额）
3. 选择默认模型 + 路由模式（自动推荐）
4. Agent 名称 + 主题色
5. 完成 → 写入配置 → 跳转聊天页

### 2.2 换模型重发 (Reroute & Resend)

**前端：**
- 每条助手消息尾部固定显示 `[🔄 换模型重发]` 按钮
- 点击 → 调用 `POST /api/agent/reroute` → 记录纠正 → 重新发送

**后端：**
- 新增 `POST /api/agent/reroute` — 接收 `{session_id, message_id, corrected_tier}`
- `services/task_router.py` 调用 `record_correction(session_id, context_hash, original_score, corrected_tier)`
- `services/correction_store.py` — SQLite 持久化，键：查询特征哈希，值：纠正记录

### 2.3 路由纠偏 (Routing Correction)

**数据存储：** `~/.hermes/hermes-webui/corrections.db` (SQLite)

**表结构：**
```sql
CREATE TABLE corrections (
    feature_hash TEXT PRIMARY KEY,
    context_fingerprint TEXT,
    original_score INTEGER,
    corrected_tier TEXT,
    count INTEGER DEFAULT 1,
    last_corrected_at TIMESTAMP
);
```

**启动时加载：** `task_router.__init__` 读取所有纠正记录，调整评分权重

**特征提取：**
- 消息长度（短/中/长）
- 是否含附件
- 是否含代码
- 是否含文件路径
- 历史用户评分低的模型
